- **分层架构**：采用经典的 Controller-Service-Model 结构。`app/api/routes.py` 处理 HTTP 请求与 SSE 进度推送；`app/services/` 封装核心业务逻辑（解析、切分、LLM 交互、组装）；`app/models/` 定义 Pydantic 数据模型。
- **流水线设计**：转换流程在后台任务中串行执行：文件解析 → 正则/启发式章节切分 → LLM 角色提取 → 带上下文记忆的逐章转换 → 剧本组装 → Schema 验证 → YAML 导出。
- **状态管理**：使用内存字典 `_jobs` 存储转换任务状态，通过 Server-Sent Events (SSE) 向客户端实时推送 `ConversionStage` 进度。
- **数据契约**：以 `app/models/screenplay.py` 中的 Pydantic 模型为单一事实来源，确保从 LLM 输出解析到最终 YAML 导出的数据结构一致性。
